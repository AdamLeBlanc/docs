Importing a site from Github

Forestry and Github are a great pair. Forestry let's you edit your content, and Github will keep a history of all your changes, as well as keep everything synced up if you make changes outside of Forestry, such as changing the theme of a site.

If you are new to git and github you can get started here, and follow an interactive tutorial here. It is very easy to learn the basics, and over time you will learn more of the advanced features that make git a powerful tool.

To import a site from github, you jut need to click on the github button when you’re setting up a new site, and select the repository and branch you want to use to import the site from. You’ll also be given a dialog asking you to signin to github and give foerstry premision to read and write to your repositores, as well as the ability add webhooks. We need the read write premision to pull sites from Github, as well as push any changes back to Github. The webhooks are needed incase you decide to make changes to your site locally (such as changing the theme) and then push to Github. Forestry will use the webhook to pull in any changes pushed to github to keep everything in sync.

GitHub Organizations

If you’re trying to connect to a repository from your GitHub organization, ensure that you have enabled 3rd-party access.

Forestry also alows you to host your sites for free on Github, with Github pages. This is a great workflow for those who are already using Github, and best of all it’s free! We have a doc here you can use to get started.
